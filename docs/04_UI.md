# Objetivo del documento

Este documento define el **sistema de diseño visual** de PsiqueMind: el lenguaje que traduce la estrategia (`BRAND`, `PRODUCT`) y la experiencia (`UX`) en decisiones concretas de color, tipografía, espacio, componentes y movimiento.

No describe qué siente el usuario (eso es `03_UX.md`).

Describe **cómo se ve y se comporta cada pixel** para que esa experiencia ocurra.

Regla de oro heredada de `BRAND.md`:

> "¿Esto ayuda a que una familia confíe en PsiqueMind?"

Si un elemento visual no aporta confianza, claridad o calma, se elimina.

---

# Principios de Diseño Visual

Seis principios gobiernan cada decisión de esta interfaz.

## 1. El aire es parte del diseño

El espacio en blanco no es vacío: es respiración, orden y lujo silencioso. Antes de agregar un elemento, preguntarse si el espacio ya estaba comunicando algo.

## 2. Calma sobre estímulo

Colores desaturados, movimiento sutil, contraste suave pero accesible. Nada compite por atención. La atención se guía, no se disputa.

## 3. Jerarquía antes que decoración

Cada pantalla tiene un único protagonista. Tamaño, peso y color sirven a la jerarquía, nunca al adorno.

## 4. La fotografía manda

Las fotos reales de la clínica son el activo emocional central. La interfaz se construye para enmarcarlas, no para taparlas.

## 5. Confianza clínica, calidez humana

Referencia visual: clínica privada moderna (Mayo Clinic, Cleveland Clinic) con la humanidad de Headspace y la limpieza de Stripe/Vercel. Profesional, nunca frío. Cálido, nunca infantil.

## 6. Accesible por defecto

WCAG AA no es una fase final: es una restricción de diseño desde el primer token. Si un color o tamaño no cumple, no entra al sistema.

## 1.6 Uso de color

- El **blanco y los neutros ocupan ~85%** de cada pantalla.
- El **azul primary** aparece en titulares y acciones — presencia, no saturación.
- El **turquesa accent** es la chispa: pequeñas dosis, alto impacto.
- **Prohibido** (por `RULES.md`): colores infantiles, saturados, contrastes agresivos, gradientes decorativos. Se permite un único gradiente **funcional**: el scrim de legibilidad sobre fotos con texto encima.

# 2. Tipografía

## 2.1 Familias

Pareja recomendada — autoridad editorial de clínica + legibilidad impecable:

| Rol | Fuente | Por qué |
|-----|--------|---------|
| Display / Titulares | **Fraunces** (serif humanista, óptica suave) | Transmite autoridad y calidez sin ser infantil ni decorativa. Diferencia del template genérico. |
| Cuerpo / UI | **Inter** (sans humanista, variable) | Hiperlegible, neutra, optimizada para web, excelente en mobile. |

Alternativa 100% sans (si se prefiere un tono más tecnológico tipo Stripe/Vercel): **Instrument Sans** para titulares + **Inter** para cuerpo.

Ambas cargan vía `font-display: swap`, subset latino, `woff2`, self-hosted para performance y privacidad. Máximo 3 pesos por familia.

Prohibido (por `RULES.md` + `BRAND.md`): tipografías infantiles, redondeadas caricaturescas o decorativas.

## 2.2 Escala tipográfica

Escala modular (tercera mayor ≈ 1.25) con `clamp()` fluido entre mobile y desktop.

| Token | Tamaño (móvil → desktop) | Line-height | Peso | Fuente |
|-------|--------------------------|-------------|------|--------|
| `display` | 2.5rem → 3.5rem | 1.1 | 500 | Fraunces |
| `h1` | 2rem → 2.75rem | 1.15 | 500 | Fraunces |
| `h2` | 1.625rem → 2rem | 1.2 | 500 | Fraunces |
| `h3` | 1.25rem → 1.5rem | 1.3 | 600 | Inter |
| `h4` | 1.125rem → 1.25rem | 1.4 | 600 | Inter |
| `body-lg` | 1.125rem → 1.125rem | 1.7 | 400 | Inter |
| `body` | 1rem → 1rem | 1.65 | 400 | Inter |
| `small` | 0.875rem | 1.5 | 400 | Inter |
| `eyebrow` | 0.8125rem | 1.4 | 600 · +8% tracking · UPPERCASE | Inter |
| `caption` | 0.75rem | 1.4 | 400 | Inter |

## 2.3 Reglas de composición

- **Medida de lectura**: 60–75 caracteres por línea (`max-width` ~65ch en bloques de texto).
- Titulares en `primary-800`; cuerpo en `neutral-800`.
- Titulares con `letter-spacing` ligeramente negativo (−0.01em); cuerpo neutro.
- Números de contadores (Certificaciones) en `display`, peso 500, `font-variant-numeric: tabular-nums`.
- Jamás justificar texto. Alineación a la izquierda por defecto; centrado solo en encabezados de sección cortos.

# 3. Espaciado y ritmo

Base **8px** con medio paso de 4px. El aire es identidad de marca.

| Token | px | Uso |
|-------|-----|-----|
| `space-1` | 4 | Micro-ajustes |
| `space-2` | 8 | Gap entre icono y texto |
| `space-3` | 12 | Padding interno compacto |
| `space-4` | 16 | Padding base |
| `space-6` | 24 | Gutter de grilla, gap de cards |
| `space-8` | 32 | Separación de bloques |
| `space-12` | 48 | Padding de cards grandes |
| `space-16` | 64 | Separación intra-sección |
| `space-24` | 96 | **Padding vertical de sección (móvil)** |
| `space-32` | 128 | **Padding vertical de sección (desktop)** |

Regla: el ritmo vertical entre secciones nunca baja de 96px en móvil / 128px en desktop. Generoso, respirado, premium.

# 5. Componentes

Cada componente cumple los tres objetivos de `BRAND.md`: transmitir confianza, guiar al usuario, mantener coherencia. Cero decoración sin propósito.

## 5.1 Botones

| Variante | Fondo | Texto | Uso |
|----------|-------|-------|-----|
| Primary | `primary-700` | `white` | Acción principal inline (contraste 9.2:1 ✔) |
| Secondary | transparente · borde `primary-700` | `primary-700` | Acción secundaria |
| Ghost | transparente | `primary-700` | Terciaria, dentro de cards |
| WhatsApp | `whatsapp` | `white` | CTA de conversión (ver 5.4) |

Especificaciones:
- Altura mínima **48px**; touch target real ≥ 44px (alcanzable con el pulgar, `UX` mobile).
- Radio `radius-md` (10px). Padding horizontal 24px.
- Peso 600, sin uppercase (calma, no grito).
- **Hover**: oscurecer un escalón (`primary-600`) + elevación `shadow-sm`, transición 150ms.
- **Active**: escala 0.98.
- **Focus-visible**: anillo `accent-600` de 2px con offset 2px. Jamás eliminar el outline.
- **Disabled**: `neutral-300`, sin sombra, cursor not-allowed.

## 5.2 Navbar

- Sticky, altura 72px (móvil) / 80px (desktop). Siempre visible (`UX`).
- **Transparente sobre el Hero → sólida (blanco + borde inferior `neutral-200` + `shadow-sm`) al hacer scroll.** Transición 250ms.
- Estructura: logo (izq) · enlaces (centro/der) · botón WhatsApp (der).
- Enlaces: `neutral-600`, hover `primary-700` con subrayado animado `accent-600` (2px, expande desde izquierda, 200ms).
- Móvil: hamburguesa → drawer full-height que abre rápido; enlaces grandes, botón WhatsApp destacado al fondo.

## 5.3 Cards (servicio, clínica, certificación)

- Superficie `white`, borde `1px neutral-200`, radio `radius-lg` (16px), padding 32px.
- **Sombra mínima**: `shadow-sm` en reposo (o solo borde). `RULES`: no abusar de sombras.
- **Hover**: elevación sutil (translateY −4px) + `shadow-md`, 200ms ease-out. Solo en cards interactivas.
- Card de servicio: icono outline `accent-600` (32px) · título `h4` · descripción `body` · responde "qué resuelve / para quién / por qué importa" (`UX`).

## 5.4 CTA de WhatsApp — decisión de diseño

El único objetivo de conversión (`PRODUCT`). Dos manifestaciones:

1. **Botón inline** en Navbar, Hero y CTA final: usa el **verde WhatsApp (`#25D366`)** con el ícono de la marca.
2. **Burbuja flotante** (esquina inferior derecha) en móvil, siempre alcanzable con el pulgar.

> **Por qué una excepción al "nada saturado":** el verde WhatsApp es un código universal de reconocimiento instantáneo. Renunciar a él por purismo cromático costaría conversión — y la conversión es el objetivo del proyecto. Se acota su uso **exclusivamente** a la acción de WhatsApp, sobre superficies blancas, con abundante aire alrededor, para que no rompa la calma general. **Validar con vos antes de fijarlo.** Alternativa conservadora: botón en `primary-700` con ícono de WhatsApp en verde.

- La burbuja no usa autoplay, sonido ni animación intrusiva (`PRODUCT` restricciones). Un solo "pulse" suave la primera vez, respetando `prefers-reduced-motion`.

## 5.5 FAQ — Acordeón

- Filas separadas por `1px neutral-200`, pregunta en `h4`, ícono +/− `accent-600` que rota 45°/180° al abrir (200ms).
- Una sola abierta a la vez. Contenido `body`, `neutral-600`.
- Accesible: `<button>` real, `aria-expanded`, navegable por teclado, foco visible (`UX` accesibilidad).

## 5.6 Proceso — Stepper (4 pasos)

Contacto → Evaluación → Diagnóstico → Tratamiento (`UX`).

- Móvil: vertical, línea conectora `accent-300` a la izquierda, número en círculo `primary-700`.
- Desktop: horizontal, 4 columnas, línea conectora entre nodos.
- Cada paso: número · título `h4` · descripción breve. Sencillo, sin sobrecarga.

## 5.7 Bloque de estadística / Contador

- Certificaciones: número grande `display` `primary-800` con **contador animado** (0 → 14+) al entrar en viewport, una sola vez, 1200ms ease-out. Etiqueta `eyebrow` debajo. `tabular-nums` para evitar salto de layout.

## 5.8 Footer

- Fondo `primary-900`, texto `neutral-100` / `neutral-300`.
- Columnas: identidad + mensaje de marca · navegación · contacto (WhatsApp `0979734997`, horario 08:00–18:00) · redes (Instagram, Facebook, TikTok, LinkedIn — íconos outline, se agregan luego).
- CTA de WhatsApp presente. Sobrio, ordenado, sin saturar.

## 5.9 Formularios (uso mínimo)

`PRODUCT` prohíbe formularios largos. Si existe algún input:
- Alto 48px, borde `neutral-300`, radio `radius-md`, foco borde `primary-500` + anillo `accent-600`.
- Label siempre visible (nunca solo placeholder). Errores en `error`, con ícono y texto, jamás solo color.

# 6. Elevación, bordes y radios

## 6.1 Sombras (uso restringido)

Solo dos niveles. Suaves, difusas, baja opacidad — nada dramático (`RULES`).

```
--shadow-sm: 0 1px 2px rgba(18,26,34,0.04), 0 2px 8px rgba(18,26,34,0.04);
--shadow-md: 0 4px 16px rgba(18,26,34,0.06), 0 8px 32px rgba(18,26,34,0.05);
```

Preferir **borde + fondo** antes que sombra para separar superficies.

## 6.2 Radios

| Token | px | Uso |
|-------|-----|-----|
| `radius-sm` | 6 | Chips, inputs pequeños |
| `radius-md` | 10 | Botones, inputs |
| `radius-lg` | 16 | Cards, imágenes |
| `radius-xl` | 24 | Contenedores destacados, Hero media |
| `radius-full` | 9999 | Burbuja WhatsApp, avatares, badges |

Radios suaves: ni cuadrado frío, ni redondeo infantil.

# 7. Iconografía

- Estilo **outline**, trazo 1.5–2px, grilla de 24px. Set consistente (Lucide o Phosphor).
- Color `accent-600` para íconos de acento; `neutral-600` para funcionales.
- Nunca rellenos caricaturescos, nunca infantiles (`RULES` + `BRAND`).
- Un solo set en todo el sitio. Coherencia total.

---

# 8. Tratamiento fotográfico

La fotografía real es un **activo estratégico** (`BRAND`, `RULES`). El diseño se adapta a la foto, no al revés.

- **Prioridad absoluta sobre stock.** Prohibido stock donde exista foto real.
- Grado de color unificado: cálido-neutro, ligeramente frío para armonizar con la paleta. Consistencia entre todas las imágenes.
- Radio `radius-lg`/`radius-xl` según contexto. `object-fit: cover`, jamás deformar.
- Aspect ratios estándar: Hero 16:9 / 4:5 (móvil), galería clínica 3:2, retrato del profesional 4:5.
- **Scrim de legibilidad** (único gradiente permitido): overlay `primary-900` de 0→55% solo donde haya texto sobre imagen, garantizando contraste AA del texto.
- `loading="lazy"`, `width`/`height` explícitos (evita CLS), formatos `AVIF`/`WebP` con fallback.

# 9. Movimiento

Las animaciones ayudan; jamás distraen (`UX`, `RULES`).

| Tipo | Duración | Easing |
|------|----------|--------|
| Micro (hover, foco) | 150ms | ease-out |
| Transición (navbar, acordeón) | 200–300ms | ease-out |
| Reveal al scroll | 500–600ms | `cubic-bezier(0.16,1,0.3,1)` |
| Contador | 1200ms | ease-out |

- **Reveal**: fade-in + translateY 16px al entrar en viewport, una sola vez.
- Scroll suave entre anclas del menú.
- **Prohibido**: parallax pesado, autoplay de video, animaciones que compitan por atención o afecten performance.
- **`prefers-reduced-motion: reduce`** → se desactivan reveals, contador salta al valor final, transiciones a 0ms. No negociable.

# 10. Accesibilidad (WCAG AA — restricción de diseño)

| Requisito | Regla del sistema |
|-----------|-------------------|
| Contraste texto normal | ≥ 4.5:1 — cuerpo `neutral-800`/blanco = ~14:1 ✔; botón blanco/`primary-700` = 9.2:1 ✔ |
| Contraste texto grande / UI | ≥ 3:1 — turquesa `accent-600` válido para íconos y texto grande, **no** para texto pequeño sobre fondo |
| Turquesa como fondo | Prohibido texto pequeño encima; usar para gráficos, líneas, íconos, o texto ≥ 24px |
| Foco | `focus-visible` anillo `accent-600` 2px, offset 2px, en todo elemento interactivo. Nunca `outline: none` sin reemplazo |
| Touch target | ≥ 44×44px (alcanzable con pulgar) |
| Color como información | Nunca único portador de significado (errores llevan ícono + texto) |
| Teclado | Navegación completa, orden lógico, skip-link al contenido |
| Semántica | HTML semántico, jerarquía de headings sin saltos, `alt` descriptivo en fotos |
| Movimiento | Respeta `prefers-reduced-motion` |

# 13. Lo que esta interfaz NUNCA hace

Heredado de `RULES.md` y `BRAND.md`:

- Parecer una plantilla o Bootstrap genérico.
- Usar colores infantiles o saturados.
- Abusar de sombras, gradientes o animaciones.
- Saturar de información o romper el aire visual.
- Usar stock donde hay foto real.
- Esconder información importante.
- Presionar, generar urgencia o miedo.
- Comprometer legibilidad, contraste o accesibilidad por estética.
